# OmniFlow Go 文档规范

更新时间：2026-04-15

适用范围：`omniflow-go` 的架构文档、进度台账、CLI 文档、后端交接说明和业务契约说明。

目标：让文档成为 Coding Agent 和开发者的外部长期记忆。文档要能回答“为什么这样做、现在事实是什么、下一步怎么改”，而不是堆积已经完成的流水账。

## 1. 文档职责

开发前必须先读相关文档，开发后必须判断是否需要更新文档。

- **开发前（Read）**：理解业务上下文、接口契约、当前状态、隐式约束。
- **开发后（Write）**：把新事实、新规则、新风险写回文档，避免下一次靠猜。

如果判断不需要更新文档，最终回复中要简短说明原因。

## 2. 目录约定

- `docs/architecture/`：长期有效的架构、分层、工程规则、开发手册。
- `docs/progress/`：阶段计划、交付状态、完成度台账。完成后应转为归档摘要，不长期保留大段流水账。
- `docs/logging.md`：日志配置、可观测性策略。
- `cmd/cli/README.md`：CLI 开发规范与命令约定。
- `docs/architecture/internal-structure-overview.md`：`internal/` 目录职责速览。

新增文档文件名使用小写英文短横线，例如 `node-dry-run-contract.md`。

## 3. 何时必须更新文档

出现以下任一情况，必须补充或更新对应文档：

- 新增或修改对外 HTTP API：路由、请求参数、响应结构、错误语义。
- 新增或修改 CLI 命令：命令名、flag、输出结构、错误语义、示例。
- 新增或修改写链路、`dry-run` 语义、事务边界或外部副作用。
- 新增或修改核心业务流程、状态流转、权限规则。
- 新增或修改日志、审计、安全、配置策略。
- 新增或修改数据库表结构、字段含义、枚举值、索引或数据演进策略。
- 新增领域概念，或修改已有术语定义。
- 修复因业务理解偏差导致的缺陷，需要澄清原有描述。
- 调整分层、目录结构、生成代码流程或交付计划。

## 4. 文档结构模板

新增长期文档优先使用以下结构，按实际需要删减：

```markdown
# 文档标题

更新时间：YYYY-MM-DD

适用范围：说明这份文档约束哪些模块或流程。

## 1. 概述

用 2～3 句话说明模块职责、边界和当前结论。

## 2. 背景

说明为什么需要该能力，面向哪些调用方或场景。

## 3. 核心概念

定义状态、枚举、术语、关键常量。

## 4. 契约

说明 HTTP、CLI、数据模型或 dry-run 契约。

## 5. 实现约束

说明分层、事务、副作用、日志、审计、测试要求。

## 6. 验证方式

说明自动化测试、手工验证方式和未覆盖风险。

## 7. 后续维护

说明哪些变更必须回写本文档。
```

阶段性进度文档完成后，改成以下精简结构：

```markdown
# 主题归档摘要

更新时间：YYYY-MM-DD
状态：已完成 / 进行中 / 暂停 / 归档

## 1. 当前结论

说明最终状态和仍然有效的事实。

## 2. 保留规则

沉淀后续仍必须遵守的工程规则。

## 3. 未完成或待验证

只保留真实未完成项，不保留已完成流水账。

## 4. 维护规则

说明什么时候需要再次更新本文档。
```

## 5. 内容要求

- 全部使用中文撰写，专业术语可保留英文。
- Markdown 层级清晰，标题不要过深。
- 表格只保留决策有用的信息，避免大面积重复“已完成”。
- 代码块标明语言类型。
- 中文与英文、数字之间保留空格。
- 涉及密钥、token、密码、内网地址等敏感信息必须使用 `***` 脱敏。
- 不把大段代码直接贴入文档，用关键规则、契约和流程描述代替。
- 不用截图替代可搜索文本。
- 不做无关台账 churn，只更新与本次变更直接相关的文档。

## 6. Agent 自检

Agent 完成一轮代码修改和测试后，必须自检：

1. 本次变更是否影响已有文档的准确性？
2. 本次变更是否引入了后续开发者、前端、CLI 使用者或 Agent 必须知道的新知识？
3. 本次变更是否涉及 HTTP API、CLI 命令、`dry-run`、日志、审计、数据库或进度台账？
4. 如果需要更新文档，是否已同步修改对应文档？
5. 如果不需要更新文档，最终回复是否说明了原因？

## 7. 禁止事项

- 禁止让已完成的进度文档长期保留大段逐批流水账。
- 禁止在文档中记录与代码事实不符的过期描述。
- 禁止把敏感配置、真实 token、密码、内部地址写入文档。
- 禁止用“后续再补文档”绕过契约、CLI、dry-run 相关变更的文档更新。
